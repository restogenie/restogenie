# **동적 데이터 매칭 UI 스토리보드**

## **Step 1: POS 연결 및 권한 승인**

1. **\[화면 진입\]** 사용자가 설정 메뉴에서 'POS 연동하기'를 클릭.  
2. **\[Vendor 선택\]** 5개 POS 로고 중 '페이히어' 클릭.  
3. **\[동적 폼 노출\]** 하단에 페이히어 연동에 필요한 'API Key' 텍스트 박스가 나타남. (만약 OKPOS 클릭 시 'ASP ID, 가맹점 코드, 비밀번호' 입력창으로 즉시 변경됨)  
4. **\[자격 증명 입력\]** 사용자가 API Key 입력 후 연결 테스트 버튼 클릭.  
5. **\[로딩 & 결과\]** 스피너가 돌며 백엔드가 1회 핑 테스트를 수행. 성공 시 "인증 성공\! 데이터를 불러옵니다." 토스트 팝업 표출 후 다음 화면 자동 전환.

## **Step 2: 동적 스키마 디스커버리 (데이터 매칭)**

1. **\[데이터 샘플링\]** 화면 상단에 "최근 50건의 트랜잭션을 분석하여 컬럼을 추출했습니다." 안내 문구.  
2. **\[그리드 렌더링\]** 중앙에 3개 열(원천 컬럼, 샘플 데이터, 타겟 컬럼)을 가진 넓은 테이블 렌더링.  
3. **\[유저 탐색\]** \- 사용자가 좌측 원천 컬럼 payment.card.amount 행을 봄.  
   * 마우스를 올리자(Hover) PreviewTooltip이 떠서 실제 JSON 응답 트리에서의 위치를 보여줌.  
4. **\[샘플 확인\]** 중앙의 샘플 데이터 영역에서 \< \> 화살표를 눌러 실제 결제 금액 5000, 12500이 들어오는 것을 시각적으로 확인.  
5. **\[오류 안내\]** 특정 행의 타겟이 '숫자'인데 샘플에 '문자열'이 섞여 있자 노란색 ⚠️ 아이콘이 뜨며 "문자열은 적재 시 제거됩니다" 안내 노출.  
6. **\[매핑 수행\]** \- 백엔드의 자동 추천 엔진에 의해 txId는 transaction\_id에 이미 매핑되어 있음 (Pre-selected).  
   * 사용자가 타겟 컬럼 SelectBox를 열어 총매출(gross\_sales\_amount), 실매출(net\_sales\_amount) 등 레스토지니 표준 스키마를 1:1로 지정.

## **Step 3: 파이프라인 가동**

1. **\[스케줄 설정\]** 테이블 하단에서 동기화 주기를 '매일 자정'으로 선택. 과거 데이터는 '최근 30일'로 선택.  
2. **\[저장\]** 매핑 저장 및 스케줄 등록 버튼 클릭.  
3. **\[백그라운드 처리\]** 프론트엔드는 현재 매핑 상태를 JSON 배열로 백엔드에 전송 \-\> 백엔드가 이를 JOLT Spec으로 컴파일하여 RDS에 저장.  
4. **\[완료\]** "데이터 연동 파이프라인이 정상적으로 가동되었습니다." 성공 화면과 함께 대시보드 홈으로 이동.
