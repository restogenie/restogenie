# **레스토지니(RestoGenie) 다중 POS 동적 데이터 매칭 서비스 기획서**

## **1\. 서비스 개요**

* **서비스명:** 레스토지니 다중 POS API 동적 데이터 매칭 및 통합 적재 파이프라인  
* **목적:** 가맹점별로 상이한 이기종 POS 시스템(토스, 페이히어, 스마트로, OKPOS, KICC 등)의 비정형 매출 데이터를 개발자 하드코딩 개입 없이 사용자가 UI를 통해 직접 표준 스키마로 매핑하고 자동 적재할 수 있는 '완전 관리형(Fully Managed) 데이터 연동 서비스' 구축.  
* **배경:** 기존 하드코딩 방식의 파이프라인은 POS 제공업체의 API 응답 구조 변경이나 신규 POS 브랜드 추가 시 유지보수 비용을 기하급수적으로 증가시킴. 이를 해결하기 위해 글로벌 제휴 마케팅 플랫폼(CJ Affiliate, AWIN 등)에서 사용하는 '메타데이터 주도(Metadata-Driven) 동적 매칭' 패러다임 도입.

## **2\. 타겟 고객 (Target Persona)**

* **프랜차이즈 본사 관리자 / 가맹점주:** 별도의 IT 부서 지원 없이 자신의 매장에서 사용하는 POS를 플랫폼에 연동하고자 하는 사용자.  
* **시스템 관리자 (내부):** 신규 POS 브랜드가 추가될 때마다 코드를 수정하는 대신, 표준 매핑 환경만 제공하여 업무 과부하를 줄이고자 하는 내부 엔지니어.

## **3\. 핵심 가치 (Core Values)**

1. **Zero Hard-Coding:** 시스템이 파싱 로직을 스스로 결정하지 않고, 사용자의 매핑 지시(Config)를 선언적 규칙(JOLT Spec)으로 변환하여 메타데이터로 관리.  
2. **동적 스키마 추론 (Dynamic Schema Inference):** 대상 POS API의 JSON 응답을 샘플링하여 계층 구조를 평탄화하고 자료형을 동적으로 파악하여 직관적인 UI로 제공.  
3. **오류 내성 (Resiliency):** 1개의 행(Row) 형식 오류로 전체 파이프라인이 중단되지 않는 행 단위 오류 처리(\_airbyte\_meta 활용) 및 지수 백오프 기반의 서드파티 API 장애 대응.  
4. **다중 테넌트 격리:** 수천 개의 매장이 동시에 동기화 작업을 수행해도 데이터 침범(Noisy Neighbor)이 발생하지 않는 Bronze-Silver-Gold 데이터 레이크 아키텍처 구현.

## **4\. 주요 기능 (Key Features)**

* **다중 인증 동적 폼:** 선택한 POS 브랜드(OAuth, API Key, ASP 인증 등)에 따라 동적으로 변하는 인증 UI 제공.  
* **실시간 프리뷰 및 데이터 샘플링:** 연동 즉시 최근 트랜잭션 5\~50건을 샘플링하여 원천 데이터 구조와 실제 값을 UI에 캐러셀 형태로 노출.  
* **1:1 컬럼 드롭다운 매핑 & 자동 매핑:** 원천 JSON 경로(Path)와 레스토지니의 글로벌 표준(ARTS ODM 기반) 스키마 간 1:1 매칭 지원 (유사도 기반 자동 추천 포함).  
* **스케줄링 기반 자동 적재:** 매핑 확정 후 사용자가 설정한 주기(예: 1시간, 자정)에 따라 AWS EventBridge를 통한 백그라운드 ETL 자동화.

## **5\. 기대 효과**

* **엔지니어링 병목 해소:** 신규 POS 연동 및 API 변경 시 백엔드 재배포 없이 DB 설정값 변경만으로 대응.  
* **데이터 품질 향상:** UI 상에서 직접 데이터를 검증하며 파이프라인 최상단에서 데이터 오염 차단.  
* **무한한 확장성:** 서버리스 기반의 다중 테넌트 아키텍처로 수만 개 가맹점까지 안정적 확장 가능.
